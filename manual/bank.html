Bank construction

1. tfr_filter_bank(tfr_array_config* a_bank_conf, tfr_filter_config* a_filter_conf)

The constructor creates an object using the structure of bank configuration for set center frequencies of filters. The second structure defines other filter parameters.

Argument:
  • a_filter_conf - pointer to a single filter configuration, frequency field in not using.
  • a_bank_conf - pointer to bank configuration.
    •
    Get configuration (methods)

    1. tfr_array_config* get_bank_configuration()
    2. tfr_array_config* get_config()
    3. tfr_filter_config* get_filter_configuration(int a_index)

    These functions return object configuration as a pointer to the bank_config type (first and second) and as a pointer to choosen tfr_filter_config. Both first and second functions work identically, thrid is different.

    Arguments and return:
  • a_index- index of choosen component filter (default value is 0)
  • return (first and second)- bank configuration,
  • return (thrid)- component filter configuration.
    •
    Filtering (methods)

    1. void filtering(complex double a_sample)
    2. tfr_complex_signal** filtering(tfr_complex_signal* a_signal, int a_hop)

    These functions realize filtering operation. The first function gets a single input sample, then result can be taken from internal filter variables m_output. The second works with the whole input signal, then output array of signals is allocated and its pointer is returned. The number of signals is equal to the number of component filters.

    Arguments and return:
  • a_sample - single input complex-valued sample,
  • a_signal - input signal,
  • a_hop - decimation factor (default value is 1),
  • return (second) - output signals.
    •
    Processed samples (method)

    1. int get_processed_sample_number()

    The function returns the number of processed samples. This number should be the same in all component filters. If not, warning is printing.

    Arguments and return:
  • return - number of processed samples.
